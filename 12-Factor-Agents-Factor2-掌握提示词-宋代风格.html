12-Factor Agents：Factor 2 - 掌握你的提示词

项目简介

12-Factor Agents 是一个开源项目，提供了构建可靠的大语言模型（LLM）应用的设计指南。这是一份专为打造生产级智能体（AI Agent）应用而编写的设计指南。

Factor 2：掌握你的提示词

本文介绍第二条 AI Agent 的设计原则：掌握你的提示词。

参考链接：⁠https://github.com/humanlayer/12-factor-agents/blob/main/content/factor-02-own-your-prompts.md

核心原则

开发者应完全掌控AI智能体的提示词，强调：⁠

  • 提示词的透明度
  • 避免依赖黑盒服务
  • 直接查看LLM框架生成的原始提示词（Prompt）
  • 不盲目依赖框架的抽象层

问题分析：依赖黑箱服务 VS 完全掌控

当前问题

当前流行的某些 Agent 框架过度封装提示词，导致提示词的"黑箱化"：⁠

  • 通过代码抽象层自动重写或构造提示词
  • 声称能"优化输出"
  • 实际隐藏了最终发送给LLM的原始提示词

过度封装的代码示例

agent = Agent(

role="...",

goal="...",

personality="...",

tools=[tool1, tool2, tool3]

)

task = Task(

instructions="...",

expected_output=OutputModel

)

result = agent.run(task)

问题影响

过度封装、黑盒化提示词往往：⁠

  • 难以精准调试或逆向拆解
  • 导致无法让 LLM 模型获得准确的令牌（tokens）

解决方案：掌控你的提示词

核心理念

开发者应真正掌控自己的提示词，将提示词视作一等代码（first-class code）来对待。

推荐方法

使用提示工程的框架或工具，自主地生成提示词，完全掌控提示词的生成。

BAML 示例

下面是通过提示工程开发语言 BAML，以函数的形式定义提示词的示例：⁠

function DetermineNextStep(thread: string) -> DoneForNow | ListGitTags | DeployBackend | DeployFrontend | RequestMoreInformation {

prompt #"

{{ _.role("system") }}

You are a helpful assistant that manages deployments for frontend and backend systems.

You work diligently to ensure safe and successful deployments by following best practices

and proper deployment procedures.

Before deploying any system, you should check:

- The deployment environment (staging vs production)

- The correct tag/version to deploy

- The current system status

You can use tools like deploy_backend, deploy_frontend, and check_deployment_status

to manage deployments. For sensitive deployments, use request_approval to get

human verification.

Always think about what to do first, like:

- Check current deployment status

- Verify the deployment tag exists

- Request approval if needed

- Deploy to staging before production

- Monitor deployment progress

{{ _.role("user") }}

{{ thread }}

What should the next step be?

"#

}

关于BAML的更多信息，可阅读：《提示工程开发语言 BAML 介绍：提示即函数，构建可维护的AI工作流》

掌控提示词的具体要求

实践要点

  • 版本控制：将提示词视为代码资产，纳入版本控制系统（如Git）
  • 配置管理：避免硬编码提示词，通过配置文件或环境变量管理
  • 质量保证：确保提示词的可测试性、可迭代性和透明度

掌控提示词的好处

核心优势

  • 完全控制：编写智能体所需的确切指令，没有黑盒抽象
  • 测试与评估：为提示词构建测试和评估，就像为任何其他代码所做的那样
  • 迭代优化：根据实际表现快速修改提示词
  • 透明度：确切知道智能体正在使用什么指令
  • 角色黑客（Role Hacking）：可以通过非标准方式操纵LLM对话角色（user/assistant）以实现非常规效果（突破标准对话角色的限制）

总结

核心观点

你的提示词（prompts）是应用程序逻辑与大语言模型（LLM）之间的核心接口。

对提示词拥有充分掌控权，能给予你在生产级智能体（production-grade agents）中所必需的：⁠

  • 灵活性
  • 提示控制力

设计哲学

我不知道什么是最好的提示词，但我知道你希望拥有能够尝试"一切可能性"的灵活性。

以上是 AI 智能体应用的设计原则"Factor 2：掌握你的提示词"的介绍。这一原则强调了在构建生产级AI应用时，开发者对提示词完全掌控的重要性。
